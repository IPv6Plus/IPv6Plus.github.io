IETF

1. Deterministic Networking Architecture

Publication URL: https://datatracker.ietf.org/doc/html/rfc8655

Introduction:

This document provides the overall architecture for Deterministic Networking (DetNet), which provides a capability to carry specified data flows for real-time applications with extremely low data loss rates and bounded latency within a network domain.

2. Deterministic Networking (DetNet) Controller Plane Framework

Publication URL: https://datatracker.ietf.org/doc/html/draft-ietf-detnet-controller-plane-framework

Introduction:

This document provides a framework overview for the DetNet controller plane. It discusses concepts and requirements that will be basis for Detnet controller plane solution documents.

3. DetNet Data Plane Framework

Publication URL: https://datatracker.ietf.org/doc/html/rfc8938

Introduction:

This document provides an overall framework for the DetNet data plane. It covers concepts and considerations that are generally common to any Deterministic Networking data plane specification.

4. Deterministic Networking (DetNet) YANG Model

Publication URL: https://datatracker.ietf.org/doc/html/draft-ietf-detnet-yang

Introduction:

This document contains the specification for Deterministic Networking flow configuration YANG Model. The model allows for provisioning of end-to-end DetNet service along the path without dependency on any signaling protocol.

5. IPv6 Options for Cyclic Queuing and Forwarding Variants

Publication URL: https://datatracker.ietf.org/doc/html/draft-yizhou-detnet-ipv6-options-for-cqf-variant

Introduction:

This document introduces a new IPv6 option to include the cycle identification to help leverage CQF variants in DetNet network to facilitate the deployments.

6. DetNet Enhanced Data Plane

Publication URL: https://datatracker.ietf.org/doc/html/draft-yzz-detnet-enhanced-data-plane

Introduction:

This document provides a method to extend DetNet data plane by introducing the Bounded Latency Information (BLI), which facilitates DetNet transit nodes to guarantee the bounded latency transmission in data plane.

7. Segment Routing for Enhanced DetNet

Publication URL: https://datatracker.ietf.org/doc/html/draft-geng-spring-sr-enhanced-detnet

Introduction:

This document defines how to leverage Segment Routing(SR) and Segment Routing over IPv6 (SRv6) to implement bounded latency.

8. ISIS-TE Extensions for Enhanced DetNet

Publication URL: https://datatracker.ietf.org/doc/html/draft-geng-lsr-isis-te-extension-enhanced-detnet

Introduction:

This document defines extensions to ISIS to distribute the enhanced DetNet information at node and/or link granularity.

9. BGP - Link State (BGP-LS) Advertisement of IGP DetNet Extensions

Publication URL: https://datatracker.ietf.org/doc/html/draft-geng-idr-bgp-ls-enhanced-detnet

Introduction:

This document defines extensions to BGP-LS to distribute the enhanced DetNet information.

10. SR Policy for enhanced DetNet

Publication URL: https://datatracker.ietf.org/doc/html/draft-zhang-sr-policy-enhanced-detnet

Introduction:

This document defines the SR policy enhancement to carry the Bounded Latency Information with a candidate path of SR policy. So that BLI behavior can be enabled automatically when the SR Policy is applied.

11. PCEP for Enhanced DetNet

Publication URL: https://datatracker.ietf.org/doc/html/draft-zhang-pce-enhanced-detnet

Introduction:

This document defines the extensions to PCEP to support the bounded-latency path computation.

12. SRv6 for Redundancy Protection

Publication URL: https://datatracker.ietf.org/doc/html/draft-ietf-spring-sr-redundancy-protection

Introduction:

This document contains the specifications of Redundancy Segment and Merging Segment for SRv6, defined to empower the Segment Routing with the capability of Redundancy Protection.

13. Redundancy Policy for Redundancy Protection

Publication URL: https://datatracker.ietf.org/doc/html/draft-geng-spring-redundancy-policy

Introduction:

This document defines redundancy policy as an extension to the current SR policy to support redundancy protection.

14. Advertising Redundancy Policy in BGP

Publication URL: https://datatracker.ietf.org/doc/html/draft-yang-idr-bgp-redundancy-policy

Introduction:

This document defines the extensions to BGP to advertise the redundancy policy.

15. Path Computation Element Communication Protocol (PCEP) Extensions to Redundancy Policy

Publication URL: https://datatracker.ietf.org/doc/html/draft-yang-pce-pcep-redundancy-policy

Introduction:

This document defines the extensions to PCEP to support the redundancy paths computation.

16. DetNet SRv6 Data Plane Encapsulation

Publication URL: https://datatracker.ietf.org/doc/html/draft-geng-detnet-dp-sol-srv6

Introduction:

This document specifies Deterministic Networking data plane operation for SRv6 encapsulated user data.

17. SRH Extension for Redundancy Protection

Publication URL: https://datatracker.ietf.org/doc/html/draft-geng-6man-redundancy-protection-srh

Introduction:

This document provides a TLV based Encapsulation of meta data used for redundancy protection, including the definitions of flow identification and sequence number.

18. IPv6 Options for DetNet

Publication URL: https://datatracker.ietf.org/doc/html/draft-pthubert-detnet-ipv6-hbh

Introduction:

This document introduces new IPv6 options that signal the path and redundancy information to the intermediate DetNet relay and forwarding nodes in IPv6 networks.

CCSA

1. The overall framework and technical requirements of deterministic IP networks for telecommunication networks

Introduction:

This is the standard project in CCSA which specifies the basic concepts, target application scenarios and requirements, technical design objectives, overall technical architecture and corresponding technical requirements of the deterministic IP network field of telecommunications networks.

2. Control plane technical requirements of Deterministic IP network for telecommunication network

Introduction:

This is the standard project in CCSA which specifies the technical requirements for the deterministic IP network control plane of telecommunications networks, and formulates the technical requirements related to the control plane of DIP, such as information notification, user information expression, configuration distribution, resource reservation, and reliability assurance, for typical DIP business models and networking forms.

3. Technical requirements for equipment in deterministic IP network for telecommunications

Introduction:

This is the standard project in CCSA which specifies the technical requirements for network equipment supporting deterministic IP capabilities in telecommunications networks, including equipment functional requirements, equipment performance requirements, operation and management requirements, environmental requirements, power supply and grounding requirements under typical application scenarios.

4. Deterministic IP Network for Telecommunications—Technical Requirements for Aggregation Layer Edge Cloud

Introduction:

This is the standard project in CCSA which specifies the concept, scope, application scenarios, and technical requirements of deterministic IP networks for aggregation layer edge clouds.

5. The service quality Indicators and Assessment Methods Of Deterministic Bearer Network

Introduction:

This is the standard project in CCSA which specifies the service quality indicators and evaluation methods of deterministic bearer networks.
