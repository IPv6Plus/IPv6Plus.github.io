IETF

Part 1: MSR6 / BIERv6

1. MSR6 Design Consideration

Publication URL: https://datatracker.ietf.org/doc/html/draft-cheng-spring-ipv6-msr-design-consideration

Introduction:

This document discusses the design consideration of IPv6 source routing multicast solution, including MSR6 Deployment Modes, Requirements and the relationship with existing IETF work.

2. MSR6 Gap Analysis

Publication URL: https://datatracker.ietf.org/doc/html/draft-li-spring-ipv6-msr-gap-analysis

Introduction:

This document analyses the gaps of the existing IPv6 multicast solutions under discussion in IETF based on the requirements..

3. RGB (Replication through Global Bitstring) Segment for MSR6

Publication URL: https://datatracker.ietf.org/doc/html/draft-lx-msr6-rgb-segment

Introduction:

This document introduces the RGB (Replication through Global Bitstring) Segment and the corresponding encapsulation for Multicast Source Routing over IPv6.

4. Source Segment for MSR6

Publication URL: https://datatracker.ietf.org/doc/html/draft-xl-msr6-source-segment

Introduction:

This document defines the general concept of source segment which is used as the IPv6 source address in an IPv6 packet. Source segment for multicast service is introduced in this document.

5. MSR6 Traffic Engineering

Publication URL: https://datatracker.ietf.org/doc/html/draft-geng-msr6-traffic-engineering

Introduction:

This document defines 2 new types of segment: End.RL and End.RL.X, and the corresponding packet processing procedures over the IPv6 data plane for the MSR6(Multicast Source Routing over IPv6) TE solutions.

6. Stateless SRv6 Point-to-Multipoint Path

Publication URL: https://datatracker.ietf.org/doc/html/draft-chen-pim-srv6-p2mp-path

Introduction:

This document describes a solution for a SRv6 Point-to-Multipoint (P2MP) Path/Tree to deliver the traffic from the ingress of the path to the multiple egresses/leaves of the path in a SR domain. There is no state stored in the core of the network for a SR P2MP path like a SR Point-to-Point (P2P) path in this solution.

7. MSR6 Side Meeting @IETF112

Publication URL: https://github.com/XuesongGeng/IETF-112-MSR6-Side-Meeting

Introduction:

MSR6 Side Meeting was to introduce the requirement for Multicast Source Routing over IPv6, including MSR6 BE, MSR6 TE, MSR6 Network Slicing and the corresponding potential solutions.

8. MSR6 BOF @IETF114

Publication URL: https://datatracker.ietf.org/doc/bofreq-liu-multicast-source-routing-over-ipv6msr6/

Introduction:

MSR6 BOF focuses on source routing multicast based on native IPv6 (IPv6 and IPv6 extension headers), including Best Effort and Traffic Engineering solutions, to decouple multicast forwarding from per-flow state at intermediate nodes.

Part 2: BIER

1. Tree Engineering for Bit Index Explicit Replication (BIER-TE)

Publication URL: https://datatracker.ietf.org/doc/html/draft-ietf-bier-te-arch

Introduction:

This document introduces per-packet stateless strict and loose path steered replication and forwarding for Bit Index Explicit Replication packets (RFC8279). This is called BIER Tree Engineering (BIER-TE).

2. BIER Fast ReRoute

Publication URL: https://datatracker.ietf.org/doc/html/draft-chen-bier-frr

Introduction:

This document describes a mechanism for fast re-route (FRR) protection against the failure of a node or link in the core of a “Bit Index Explicit Replication” (BIER) domain.

Part 3: SR Multicast

1. SR Replication Segment for Multi-point Service Delivery

Publication URL: https://datatracker.ietf.org/doc/html/draft-ietf-spring-sr-replication-segment

Introduction:

This document describes the SR Replication segment for Multi-point service delivery. A SR Replication segment allows a packet to be replicated from a Replication Node to downstream nodes.

2. Segment Routing Point-to-Multipoint Policy

Publication URL: https://datatracker.ietf.org/doc/html/draft-ietf-pim-sr-p2mp-policy

Introduction:

This document describes an architecture to construct a Point-to-Multipoint (P2MP) tree to deliver Multi-point services in a Segment Routing domain. A SR P2MP tree is constructed by stitching a set of Replication segments together. A SR Point-to-Multipoint (SR P2MP) Policy is used to define and instantiate a P2MP tree which is computed by a PCE.

3. Multicast and Ethernet VPN with Segment Routing Point-to-Multipoint Trees

Publication URL: https://datatracker.ietf.org/doc/html/draft-ietf-bess-mvpn-evpn-sr-p2mp

Introduction:

A Point-to-Multipoint (P2MP) Tree in a Segment Routing domain efficiently carries traffic from a Root to a set of Leaves. This document describes extensions to BGP encodings and procedures for P2MP trees used in BGP/MPLS IP VPNs and Ethernet VPNs.

4. YANG Data Model for p2mp sr policy

Publication URL: https://datatracker.ietf.org/doc/html/draft-hb-spring-sr-p2mp-policy-yang

Introduction:

This document defines a YANG data model for SR P2MP Policy Configuration and operation.

5. PCEP extensions for p2mp sr policy

Publication URL: https://datatracker.ietf.org/doc/html/draft-ietf-pce-sr-p2mp-policy

Introduction:

SR P2MP policies are set of policies that enable architecture for P2MP service delivery. This document specifies extensions to the Path Computation Element Communication Protocol (PCEP) that allow a stateful PCE to compute and initiate P2MP paths from a Root to a set of Leaves.

6. Advertising p2mp policies in BGP

Publication URL: https://datatracker.ietf.org/doc/html/draft-hb-idr-sr-p2mp-policy

Introduction:

This document specifies a new BGP SAFI with a new NLRI in order to advertise P2MP policy from a controller to a set of nodes.

7. P2MP Policy Ping

Publication URL: https://datatracker.ietf.org/doc/html/draft-hb-pim-p2mp-policy-ping

Introduction:

This document describes a simple and efficient mechanism that can be used to detect data plane failures in P2MP Policy Candidate Paths (CPs) and Path Instances (PIs).

Part 4: PCECC for Multicast

1. PCECC for Multicast

Publication URL: https://datatracker.ietf.org/doc/html/draft-dhody-pce-pcep-extension-pce-controller-p2mp

Introduction:

This document specifies the procedures and Path Computation Element Communication Protocol (PCEP) extensions for using the PCE as the central controller for provisioning labels along the path of the static P2MP LSP.
