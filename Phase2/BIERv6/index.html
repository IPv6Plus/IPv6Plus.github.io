IETF

Part 1: BIERv6

1. BIER IPv6 Requirements

Publication URL: https://tools.ietf.org/html/draft-ietf-bier-ipv6-requirements

Introduction:

This document describes the BIER IPv6 requirements, summarize the proposed solutions, and guide the working group in understanding the benefits, and drawbacks, of the various solutions and to help in the development of acceptable solutions.

2. Encapsulation for BIER in Non-MPLS IPv6 Networks

Publication URL: https://tools.ietf.org/html/draft-xie-bier-ipv6-encapsulation

Introduction:

This document proposes a BIER IPv6 (BIERv6) encapsulation for Non-MPLS IPv6 Networks using the IPv6 Destination Option extension header.

3. Use of BIER IPv6 Encapsulation (BIERv6) for Multicast VPN in IPv6 networks

Publication URL: https://tools.ietf.org/html/draft-xie-bier-ipv6-mvpn

Introduction:

This document defines the procedures and messages for using Bit Index Explicit Replication (BIER) for Multicast VPN Services in IPv6 networks using the BIER IPv6 encapsulation.

4. BIER IPv6 Encapsulation (BIERv6) Support via IS-IS

Publication URL: https://tools.ietf.org/html/draft-xie-bier-ipv6-isis-extension

Introduction:

This document defines IS-IS extensions to support multicast forwarding using the Bit Index Explicit Replication (BIER) with IPv6 encapsulation (BIERv6).

5. YANG Data Model for Bierv6

Publication URL: https://tools.ietf.org/html/draft-geng-bier-bierv6-yang

Introduction:

This draft defines a YANG data model for Bierv6 (Bit indexed explicit replication over IPv6).

6. Inter-Domain Multicast Deployment using BIERv6

Publication URL: https://tools.ietf.org/html/draft-geng-bier-ipv6-inter-domain

Introduction:

This document introduces the techniques for multicast deployment across multiple domains using BIERv6, and demonstrate how BIERv6 is beneficial for such deployment.

7. Tree Engineering for Bit Index Explicit Replication (BIER-TE)

Publication URL: https://datatracker.ietf.org/doc/html/draft-ietf-bier-te-arch

Introduction:

This document introduces per-packet stateless strict and loose path steered replication and forwarding for Bit Index Explicit Replication packets (RFC8279). This is called BIER Tree Engineering (BIER-TE).

8. BIER Fast ReRoute

Publication URL: https://datatracker.ietf.org/doc/html/draft-chen-bier-frr

Introduction:

This document describes a mechanism for fast re-route (FRR) protection against the failure of a node or link in the core of a “Bit Index Explicit Replication” (BIER) domain.

Part 2: SR Multicast

1. SR Replication Segment for Multi-point Service Delivery

Publication URL: https://datatracker.ietf.org/doc/html/draft-ietf-spring-sr-replication-segment

Introduction:

This document describes the SR Replication segment for Multi-point service delivery. A SR Replication segment allows a packet to be replicated from a Replication Node to downstream nodes.

2. Segment Routing Point-to-Multipoint Policy

Publication URL: https://datatracker.ietf.org/doc/html/draft-ietf-pim-sr-p2mp-policy

Introduction:

This document describes an architecture to construct a Point-to-Multipoint (P2MP) tree to deliver Multi-point services in a Segment Routing domain. A SR P2MP tree is constructed by stitching a set of Replication segments together. A SR Point-to-Multipoint (SR P2MP) Policy is used to define and instantiate a P2MP tree which is computed by a PCE.

3. Multicast and Ethernet VPN with Segment Routing Point-to-Multipoint Trees

Publication URL: https://datatracker.ietf.org/doc/html/draft-ietf-bess-mvpn-evpn-sr-p2mp

Introduction:

A Point-to-Multipoint (P2MP) Tree in a Segment Routing domain efficiently carries traffic from a Root to a set of Leaves. This document describes extensions to BGP encodings and procedures for P2MP trees used in BGP/MPLS IP VPNs and Ethernet VPNs.

4. YANG Data Model for p2mp sr policy

Publication URL: https://datatracker.ietf.org/doc/html/draft-hb-spring-sr-p2mp-policy-yang

Introduction:

This document defines a YANG data model for SR P2MP Policy Configuration and operation.

5. PCEP extensions for p2mp sr policy

Publication URL: https://datatracker.ietf.org/doc/html/draft-hsd-pce-sr-p2mp-policy

Introduction:

SR P2MP policies are set of policies that enable architecture for P2MP service delivery. This document specifies extensions to the Path Computation Element Communication Protocol (PCEP) that allow a stateful PCE to compute and initiate P2MP paths from a Root to a set of Leaves.

6. Advertising p2mp policies in BGP

Publication URL: https://datatracker.ietf.org/doc/html/draft-hb-idr-sr-p2mp-policy

Introduction:

This document specifies a new BGP SAFI with a new NLRI in order to advertise P2MP policy from a controller to a set of nodes.

7. P2MP Policy Ping

Publication URL: https://datatracker.ietf.org/doc/html/draft-hb-pim-p2mp-policy-ping

Introduction:

This document describes a simple and efficient mechanism that can be used to detect data plane failures in P2MP Policy Candidate Paths (CPs) and Path Instances (PIs).

8. Stateless SRv6 Point-to-Multipoint Path

Publication URL: https://datatracker.ietf.org/doc/html/draft-chen-pim-srv6-p2mp-path

Introduction: This document describes a solution for a SRv6 Point-to-Multipoint (P2MP) Path/Tree to deliver the traffic from the ingress of the path to the multiple egresses/leaves of the path in a SR domain. There is no state stored in the core of the network for a SR P2MP path like a SR Point-to-Point (P2P) path in this solution.

1. BIERv6 Test Report White Paper [EN]

Introduction:

China Unicom Beijing branch, in conjunction with Huawei, has tested BIERv6 multicast solution with the IPTV service in existing metro network. This test is an end-to-end verification of the actual IPTV video service, proving that BIERv6 solution has the advantages of easy deployment, on-demand access, high reliability and intelligent operation and maintenance. This white paper covers the topology, network device and the features that have been tested.
